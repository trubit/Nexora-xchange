◈ Step 6 — Expected "Normal" States
These are not errors — they are correct behavior for a freshly started system with no live market connections.
✓
Blockchain chains show "disconnected"
Ethereum, BSC, Polygon, Arbitrum, Optimism RPC nodes are not configured in .env. The settlement service starts but idles. This is correct — BLOCKCHAIN_ENABLED=false.
✓
Redis unavailable messages in server log
EventBus, MarketLock, and Notifications fall back to in-process mode when Redis is not running. The exchange works fully without Redis in development.
✓
Empty tables on all new pages
Arbitrage, Liquidity, Market Intelligence, Execution Router, and Audit Ledger show empty states when no data has been generated yet. The pages are correctly wired — they just need real trading activity to populate.
✓
SMTP error in server log
Brevo SMTP verification fails on startup because the server IP is not whitelisted in the Brevo dashboard. Email sending is non-critical — login, trading, and all enterprise features work independently.
✓
ECONNREFUSED on socket.io at startup
The browser opens before the Express server finishes its ~13-second startup. Socket.io retries automatically — these errors clear on their own within 30 seconds.
